# ZIP Health Atlas — web app

A polished, **static**, accessible atlas of U.S. health outcomes by ZIP code. A MapLibre + PMTiles
choropleth (recolored via feature-state, never a source rebuild) with four D3 analytical panels, an
insight rail, a ZIP detail card, and URL-shareable state.

## Architecture

- **Static export** (`next build` → `out/`). No server, no secrets in the bundle. The browser fetches
  precomputed JSON from `public/data/` and streams the public PMTiles over HTTPS range requests.
- **Map**: `components/MapChoropleth.tsx` + `useFeatureStateMetric.ts`. The PMTiles vector source uses
  `promoteId: "zip_code"`. Each metric pushes two feature-state fields per ZIP — `val` (raw) and `pct`
  (national percentile). Switching **mode** (rate / gap / percentile) swaps only the `fill-color` paint
  expression; switching **metric** re-pushes feature-state. The source is never rebuilt and tiles are
  never refetched on a switch.
- **Charts**: D3-first (`components/panels/*`, scales in `useMemo`, marks as JSX, axes as refed islands).
  Each panel ships a table fallback and shows values without hover.
- **Color** (`lib/colors.ts`): one sequential burden ramp (rate/percentile) and one diverging ramp (gap,
  midpoint = national benchmark), shared identically by the map and the panels. Selection is encoded by
  weight + halo + label, never by color alone. Explicit no-data color.
- **State** (`lib/urlState.ts`): `metric`, `mode`, `region`, `selected` live in the URL — copy-link
  reproduces a view and the back button steps through it.
- **Data contract**: [`../docs/data-contract.md`](../docs/data-contract.md). Payloads are generated by
  [`../data-prep/prep.py`](../data-prep/prep.py) into `public/data/` and verified by `../data-prep/qa.py`.

## Run

```bash
npm install
npm run dev          # http://localhost:3000
```

## Build (static)

```bash
npm run build        # outputs to ./out  (output: 'export')
npx serve out        # preview the static export, or deploy ./out to any static host / Vercel
```

The app needs no environment variables at runtime. The Tigris S3 credentials in the repo root `.env`
are used **only** by the build-time prep pipeline (`../data-prep/`); they never reach `web/` or the client.

## Regenerate data payloads

```bash
cd ../data-prep
uv venv .venv && uv pip install --python .venv -r requirements.txt
.venv/Scripts/python.exe extract_pmtiles.py   # decode 15 health measures from the PMTiles
.venv/Scripts/python.exe prep.py              # emit web/public/data/*
.venv/Scripts/python.exe qa.py                # independent reconciliation vs raw
```

## Test

```bash
npm test                       # vitest unit tests (URL codec, color ledger, formatters)
npx playwright test            # visual-regression baselines + e2e + a11y
npx playwright test -u         # update visual baselines after intentional UI changes
```

Visual-regression baselines live in `tests-e2e/visual.spec.ts-snapshots/`. The map's WebGL layer is
masked in screenshots (it does not composite in headless Chromium and is inherently volatile); baselines
cover the stable UI chrome, controls, legend, insight rail, and the four D3 panels.

## Accessibility

Keyboard-operable controls and chart marks with visible focus; SR text alternatives + per-chart table
fallbacks; values visible without hover; nothing encoded by color alone; `prefers-reduced-motion`
honored (instant transitions, no map fly animation); explicit no-data styling.
